# Niz_Week4_Summary

### 개념 정리

- **Node.js:**
  브라우저 환경 밖에서도 자바스크립트를 실행할 수 있게 해주는 '자바스크립트 런타임(실행 환경)'.
- **엔진과 라이브러리:**
  - **V8 엔진:** 구글이 개발한 엔진으로,
    자바스크립트 코드를 해석하고 최적화된 머신 코드로 컴파일하여 실행함.
  - **libuv 라이브러리:** C 라이브러리로, 운영체제의 내부 API와 소통하며
    파일 시스템이나 네트워크 등의 작업을 비동기적으로 처리할 수 있게 도움.
- **작동 방식 (싱글 스레드 + 논블로킹 I/O):**
  - Node.js는 자바스크립트를 실행하는 메인 스레드가 단 1개뿐임 (싱글 스레드).
  - BUT I/O(입출력) 요청이 들어오면 백그라운드로 넘기고(논블로킹),
    메인 스레드는 멈추지 않고 바로 다음 코드를 실행함.
  - 백그라운드 작업이 끝나면 이벤트 루프(Event Loop)가 스케줄러 역할을 하여
    완료된 작업을 큐에서 꺼내 메인 스레드로 다시 전달함.
- **장점**: 크기는 작지만 개수가 많은 데이터를 실시간으로 처리해야 하는 I/O 중심 작업(API )에 효율적임.
- **단점**: 스레드가 하나뿐이므로 영상 인코딩 같은 복잡하고 무거운 CPU 연산 작업에는 취약함.

### 실습 코드

- **TypeScript 제네릭(Generic)**

```tsx
// 제네릭을 활용한 배열의 첫 번째 요소 반환 함수
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

const i = getFirst([1, 2, 3]); // i = 1
const j = getFirst(["U", "M", "C"]); // j = "U"
```

- 만약 제네릭이 없다면, 숫자 배열을 위한 `getFirstNumber`, 문자열 배열을 위한 `getFirstString` 등
  타입이 바뀔 때마다 동일한 로직의 함수를 수백 개씩 만들어야 함.
- 꺾쇠 `<T>` : “지금 당장 타입을 정하지 않고, 함수를 쓸 때 들어오는 값에 따라 타입을 정하겠다"는 의미.

**→** 숫자가 들어오면 `T`가 자동으로 `number`가 되고, 문자열이 들어오면 `string`이 됨.
결과적으로 단 3줄의 코드로 모든 타입의 배열에 대응할 수 있는 강력한 범용 함수가 됨.

### 실습 정리

- **디렉토리 모듈화:** 비즈니스 도메인(예: users)별로 모듈 폴더를 생성하고, 그 내부에 다시 `controllers`, `services`, `repositories`, `dtos` 폴더를 생성하여 레이어를 분리함.
  → 기능 단위로 코드를 쉽게 찾고 관리할 수 있음.
- **환경 변수 (`.env`) 보호:**
  - 포트 번호(PORT)나 데이터베이스 접속 URL과 같이 외부에 노출되면 안 되는 민감한 정보는
    소스코드에 직접 적지 않고 `.env` 파일에 분리하여 저장함.
  - **주의점:** 깃허브 등 공개 저장소에 이 정보가 올라가는 것을 막기 위해, 프로젝트를 시작할 때 `.gitignore` 파일에 반드시 `.env`를 추가해 두어야 함.
